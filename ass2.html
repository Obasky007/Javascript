JavaScript

let vs var vs const

JavaScript has three main ways to declare variables: var, let, and const. They are similar in that all are used to store values, but they differ in scope, re-declaration, re-assignment, and hoisting.

Key Differences

  • var → Function-scoped, can be re-declared & re-assigned, hoisted with undefined.
  • let → Block-scoped, cannot be re-declared, can be re-assigned, hoisted but not initialized (TDZ).
  • const → Block-scoped, cannot be re-declared or re-assigned, must be initialized, also has TDZ.

Similarities

  • All three are used to declare variables.
  • let and const were introduced in ES6 (modern JavaScript).
  • let and const both provide block scope and avoid common var pitfalls.

1. Scope

if (true) {
var a = 1; // function scope
let b = 2; // block scope
const c = 3; // block scope
}
console.log(a); // works
console.log(b); // error
console.log(c); // error

2. Redeclaration & Reassignment

var x = 1; var x = 2; // allowed
let y = 3; // let y = 4; not allowed
y = 5; // reassignment allowed
const z = 6; // const z = 7 no redeclare
z = 8; // no reassignment

3. Hoisting

console.log(a); // undefined ( var hoisted)
var a = 10;

console.log(b); // ReferenceError
let b = 20;

console.log(c); // ReferenceError
const c = 30;
